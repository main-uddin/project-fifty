4️⃣ Events & event flow (VERY IMPORTANT 🔥)
APIs

event.stopPropagation // done

What is stopPropagation()? (Interview Answer)
event.stopPropagation() stops an event from propagating further through the DOM, meaning it prevents the event from reaching parent elements during capturing or bubbling, but it does not prevent the browser’s default behavior.

event.preventDefault // done

What is preventDefault()? (Interview Answer)
event.preventDefault() prevents the browser’s default behavior for an event, but it does NOT stop event propagation.

Concepts (more important than APIs)

event bubbling // done

Event Bubbling (Interview Answer)
Event bubbling means that when an event happens on an element, it first runs on that element and then propagates upward through its parent elements until it reaches the document.

capturing // done

Event Capturing (Interview Answer)
Event capturing is the opposite of bubbling. The event starts from the top of the DOM tree and travels down to the target element. Only handlers registered with capture: true run in this phase.

event delegation // done

🔹 Event Delegation (Interview Answer)
Event delegation is a pattern where a single parent element handles events for its child elements by relying on event bubbling and checking event.target.
